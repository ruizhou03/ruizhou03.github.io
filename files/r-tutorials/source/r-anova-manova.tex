\hypertarget{nested-anova-1}{%
\subsection{nested ANOVA}\label{nested-anova-1}}

Theoretically, the order of computing the variance sum of squares
matters, thus appointing the
\texttt{summation\ =\ \textquotesingle{}I\textquotesingle{}}.

However, here the summation type does not matter.

Note that the result of the main effect of the group is wrong. We can
manually make it straight from the SS of the group and the subgroup.

Using the Box's M-test and Levene's Test.

\hypertarget{pipe-friendly-preferred-1}{%
\paragraph{pipe-friendly (Preferred)}\label{pipe-friendly-preferred-1}}

Using \texttt{box\_m()} from the package \texttt{rstatix} to test the
homogeniety of covariance matrices, in which the first parameter is the
DVs needed to be analyzed (No group variable! This can be achieved
through \texttt{select()}), and the second is the \texttt{group}
parameter specifying the group variable.

Moreover, the function \texttt{heplots::boxM()} is helpful. Instead of
taking the formula to specify the parameters, using the ``traditional''
way to write \texttt{group\ =\ ...} works pretty well combined with
pipe-friendly environment!

The drawback of the functions is that they could not search column
variables automatically, and that should be specified manually. And more
annoying one is that, when encountering with NAs, an error will be
raised by \texttt{rstatix::box\_m()}, while that's fine with
\texttt{heplots::boxM()}.

\hypertarget{effect-size}{%
\subsection{effect size}\label{effect-size}}

Checking the effect size is also indispensable.

Use \texttt{eta\_squared()} from the package \texttt{effectsize}, which
contains plenty of useful functions computing effect size under various
scenarios.

Note that there's also an \texttt{eta\_squared()} from the package
\texttt{rstatix()}, but that can only deal with univariate ANOVA, it
would fail when encountering MANOVA models.

Also note that the package \texttt{effectsize} is incorporated in
\texttt{bruceR}!

\hypertarget{post-hoc-tests}{%
\subsection{post-hoc tests}\label{post-hoc-tests}}

Note that the post-hoc tests here are essentially for the univariate
ANOVA.

Note that the \texttt{games\_howell\_test()} is different from
\texttt{tukey\_hsd()}.

Note that for many DVs to take the post-hoc tests simultaneously, first
\texttt{gather()} the targeted columns, and then take the
\textbf{grouped} data for the test.
