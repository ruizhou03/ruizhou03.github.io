机票监控 · 自带运行器

把盯票装到你自己电脑上

照着做，全程复制粘贴、不用改任何代码。配好后它每天替你盯票，便宜票一出现就按你选的方式喊你。目前支持 macOS，Windows / Linux 版在做。

你的模式
完整模式：用 AI 判断哪张票值得抢、做价格趋势分析，并给你发邮件简报。需要你选的 AI 厂商的 API key 和一个邮箱应用密码（下面手把手教，约 5 分钟）。 简单模式：只按你设的价格阈值盯票，结果显示在本地面板里、并弹桌面通知。不需要任何密钥、不用配邮箱，最快上手。随时可升级到完整模式。

  1. 装个底座：Python

    运行器用 Python 跑。先看看你电脑有没有——打开终端（聚焦搜 终端），粘贴：

    python3 --version

    看到 Python 3.10 以上就行。没有的话去 python.org 下载安装，或 brew install python。

  2. 拿到「你的」安装命令

    回到配置页，确认设置无误后点 「生成我的盯票」。网站会给你一行已经带好你配置的安装命令——直接复制（这里是示意）：

    curl -fsSL https://ruizhou03.com/flight/get | bash # 末尾会自动带上你的盯票编号

    你也可以在配置页点「下载配置文件」，把 flightwatch.json 存到下载夹——安装器会自动找到它。整个配置只是一份数据，不含任何代码。

  3. 运行它

    把上一步那行粘进终端、回车。它会自动装好浏览器内核和依赖，然后一步步问你几个问题：

    [[zi:check]] 正在安装盯票运行器…（约 1–2 分钟） [[zi:check]] 已读取你的盯票配置：你的出发地 → 你的目的地 ? 选择模式： › 完整（AI+邮件） ? AI 模型（FLIGHTWATCH_MODEL）： anthropic:claude-opus-4-8 ? 粘贴该厂商的 API key： **************** ? 收件邮箱： you@gmail.com ? 邮箱应用密码（16 位）： **************** ? 选择模式： › 简单（价格阈值，无需密钥） ? 结果怎么看： › 本地面板 + 桌面通知 [[zi:check]] 已设好定时：每天 10:00 / 22:00 自动盯票 [[zi:check]] 安装完成！正在跑第一轮抓取（约 15–30 分钟，期间无输出属正常，别关窗口）…

    简单模式到这里就装完了，没有下面要密钥的两步——可以直接跳到最后一步。

  4. 选 AI 厂商、领 key、定模型 仅完整模式

    完整模式让 AI 判断哪张票值得抢、估触发价、写简报。你用任意主流厂商都行——Claude / GPT / DeepSeek / Kimi / GLM…——用你自己的 key。下面选好厂商和模型，得到一个 FLIGHTWATCH_MODEL 值，安装器问到时连同 key 一起粘进去。

    1. 选一个你有账号的厂商 → 去它官网领 API key（充一点额度）。
    2. 在它的官方模型列表查当前模型 ID，填到下面。
    模型列表 →
    FLIGHTWATCH_MODEL=anthropic:claude-opus-4-8

    各厂商模型 ID 随时更新，务必以官方文档为准（Claude 的下拉是最新；其它厂商点上面链接查当前 ID）。

    [[zi:bulb]] 用量很省：每天 2 轮、判断几十条航班，通常一个月几美元（具体看你选的厂商 / 模型定价）。

  5. 生成一个邮箱应用密码 仅完整模式

    这样运行器才能用你的邮箱给你发盯票简报。不是你的登录密码，是一串专用的 16 位码（以 Gmail 为例）：

    1. 打开 Google 账号 → 安全性，先开启 两步验证（必须先开才有下一步）。
    2. 搜索 「应用专用密码 / App passwords」 → 取个名字（如 flightwatch）→ 生成。
    3. 把那 16 位码复制，粘到安装器问「邮箱应用密码」处。
    用别的邮箱（Outlook/QQ/163）也行，原理一样：开 SMTP / 拿授权码。手册里会按你填的邮箱给对应指引。

  6. 完成 —— 它已经开始替你盯了

    首轮跑完（约 15–30 分钟后）你会收到一封 「已开始盯票」确认邮件；之后每天 10:00 / 22:00 自动跑，捡到便宜票就给你发那张机票卡。

    装好后会弹一条 「已开始盯票」桌面通知；之后每天自动跑，结果都在本地面板里，便宜票会再弹通知。打开面板：flightwatch panel。

    真正在跑的全过程都在你自己电脑上，你的出行计划、邮箱、key 都不经过任何服务器，也不进任何公开仓库。

之后想改 / 排错

想改航线、价格或优先级怎么办？

回配置页改好 → 再点「生成我的盯票」→ 把新配置文件拖进去（或重跑那行命令）。设置是数据，随时换，不动代码。

怎么确认它在正常跑？

命令 flightwatch status 看上次运行时间和结果；完整模式每天都会来信（哪怕没便宜票也有「盯票正常」简报）；本地面板里有运行日志。

想暂停或卸载？

暂停：flightwatch pause；恢复：flightwatch on；彻底卸载：flightwatch uninstall，一条命令清干净，不留垃圾。

电脑关机 / 睡眠时会漏跑吗？

大多不会漏：开机或从睡眠唤醒时，若还在当天的盯票时段窗口内（约上午 10–14 点 / 晚间 22–24 点）会自动补跑那一轮；若唤醒时已错过该窗口，则顺延到下一时段。电脑长期关机自然就不盯了——这是「自带运行器」跑在你机器上的本质。

把配置发给朋友，他也能用吗？

能。你的「配置码」纯数据可分享，他在自己电脑上照这份手册装一遍即可（完整模式各自用自己的 key、互不影响）。
